# 인프라 설정

## 필요한 인프라 구성 파일

부하 테스트를 위한 인프라를 설정하기 위해 다음 파일들이 필요합니다:

### 1. 프로메테우스 설정

**파일 경로**: `infrastructure/prometheus/prometheus.yml`

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: "prometheus"
    static_configs:
      - targets: ["localhost:9090"]

  - job_name: "k6"
    static_configs:
      - targets: ["k6:6565"]

  - job_name: "gateway"
    static_configs:
      - targets: ["gateway:3000"]

  - job_name: "auth"
    static_configs:
      - targets: ["auth:3001"]

  - job_name: "event"
    static_configs:
      - targets: ["event:3002"]
```

### 2. k6 출력 설정

**파일 경로**: `test/config/k6-output.json`

```json
{
  "prometheus": {
    "pushgateway": "http://prometheus:9091",
    "insecureSkipTLSVerify": true,
    "enableExemplarLabels": true
  }
}
```

### 3. Docker Compose 설정 업데이트

**파일 경로**: `docker-compose.k6.yml`

```yaml
version: "3.9"

services:
  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./infrastructure/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus-data:/prometheus
    command:
      - "--config.file=/etc/prometheus/prometheus.yml"
      - "--storage.tsdb.path=/prometheus"
      - "--web.console.libraries=/etc/prometheus/console_libraries"
      - "--web.console.templates=/etc/prometheus/consoles"
      - "--web.enable-lifecycle"
    networks:
      - event-network

  k6:
    image: grafana/k6:latest
    container_name: k6
    ports:
      - "6565:6565"
    volumes:
      - ./test:/test
      - ./dist:/dist
    environment:
      - K6_PROMETHEUS_RW_SERVER_URL=http://prometheus:9090/api/v1/write
      - K6_OUT=output-prometheus-remote
    networks:
      - event-network
    depends_on:
      - prometheus

volumes:
  prometheus-data:

networks:
  event-network:
    external: true
```

### 2. 데이터소스 구성

**파일 경로**: `infrastructure/grafana/datasources/prometheus.yaml`

```yaml
apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: true
```

### 3. 대시보드 프로비저닝 구성

**파일 경로**: `infrastructure/grafana/dashboards.yaml`

```yaml
apiVersion: 1

providers:
  - name: "k6-dashboards"
    orgId: 1
    folder: "k6"
    type: file
    disableDeletion: false
    updateIntervalSeconds: 10
    allowUiUpdates: true
    options:
      path: /etc/grafana/dashboards
      foldersFromFilesStructure: true
```

## 시스템 메트릭 수집을 위한 cAdvisor 설정

**파일 경로**: `docker-compose.k6.yml` (추가)

```yaml
cadvisor:
  image: gcr.io/cadvisor/cadvisor:latest
  container_name: cadvisor
  volumes:
    - /:/rootfs:ro
    - /var/run:/var/run:ro
    - /sys:/sys:ro
    - /var/lib/docker/:/var/lib/docker:ro
    - /dev/disk/:/dev/disk:ro
  ports:
    - "8080:8080"
  networks:
    - event-network
  depends_on:
    - prometheus
```

## 완성된 모니터링 스택

이 설정을 통해 다음과 같은 모니터링 스택이 구성됩니다:

1. **k6**: 부하 테스트 도구
2. **Prometheus**: 메트릭 수집 및 저장
3. **Grafana**: 대시보드 시각화
4. **cAdvisor**: 컨테이너 리소스 모니터링

모든 구성 요소는 Docker Compose를 통해 연결되며, 테스트 실행 중 및 완료 후에 Grafana 대시보드에서 결과를 확인할 수 있습니다.
